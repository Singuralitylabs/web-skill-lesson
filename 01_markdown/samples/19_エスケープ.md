# エスケープ サンプル

## 解説

Markdownの記号をそのまま文字として表示したいときに使います。
バックスラッシュ `\` を記号の前に付けます。

**対応環境**: すべてのMarkdown環境

---

## 基本的なエスケープ

### 記法

```markdown
\*アスタリスク\*
\# シャープ
\`バッククォート\`
\- ハイフン
\. ピリオド
```

### 表示結果

\*アスタリスク\*
\# シャープ
\`バッククォート\`
\- ハイフン
\. ピリオド

---

## エスケープが必要な記号一覧

| 記号 | 名前 | Markdown での意味 | エスケープ |
|:----:|------|------------------|------------|
| `\` | バックスラッシュ | エスケープ文字 | `\\` |
| `` ` `` | バッククォート | インラインコード | `` \` `` |
| `*` | アスタリスク | 強調（太字/斜体） | `\*` |
| `_` | アンダースコア | 強調（太字/斜体） | `\_` |
| `#` | シャープ | 見出し | `\#` |
| `+` | プラス | リスト | `\+` |
| `-` | ハイフン | リスト/水平線 | `\-` |
| `.` | ピリオド | 番号付きリスト | `\.` |
| `!` | エクスクラメーション | 画像 | `\!` |
| `[` `]` | 角括弧 | リンク | `\[` `\]` |
| `(` `)` | 丸括弧 | リンクURL | `\(` `\)` |
| `{` `}` | 波括弧 | 一部の拡張記法 | `\{` `\}` |
| `<` `>` | 山括弧 | HTML/自動リンク | `\<` `\>` |
| `|` | パイプ | 表 | `\|` |
| `~` | チルダ | 打ち消し線 | `\~` |

---

## 実用例

### アスタリスクを表示

エスケープなし：
```markdown
*これは斜体になります*
```
表示：*これは斜体になります*

エスケープあり：
```markdown
\*これはそのまま表示\*
```
表示：\*これはそのまま表示\*

---

### シャープを表示

エスケープなし：
```markdown
# これは見出しになります
```

エスケープあり：
```markdown
\# これはそのまま表示されます
```
表示：\# これはそのまま表示されます

---

### 番号付きリストを防ぐ

エスケープなし：
```markdown
1. これはリストになります
```
表示：
1. これはリストになります

エスケープあり：
```markdown
1\. これはそのまま表示されます
```
表示：1\. これはそのまま表示されます

---

### 表内でパイプを表示

エスケープなし：`|` は表のセル区切りになる

エスケープあり：
```markdown
| コマンド | 説明 |
|----------|------|
| `a \| b` | aまたはb |
| `grep "foo\|bar"` | fooまたはbar を検索 |
```

| コマンド | 説明 |
|----------|------|
| `a \| b` | aまたはb |
| `grep "foo\|bar"` | fooまたはbar を検索 |

---

### バックスラッシュ自体を表示

```markdown
Windowsのパス: C:\\Users\\username
エスケープ記号: \\
```

表示：
Windowsのパス: C:\\Users\\username
エスケープ記号: \\

---

## よくある使用場面

### 技術文書での記号説明

```markdown
## Markdownの記法

- 見出し: \# を使用
- 太字: \*\*テキスト\*\* または \_\_テキスト\_\_
- リスト: \- または \* を行頭に
```

表示：

- 見出し: \# を使用
- 太字: \*\*テキスト\*\* または \_\_テキスト\_\_
- リスト: \- または \* を行頭に

---

### 正規表現の説明

```markdown
## 正規表現パターン

- `\d` : 数字にマッチ
- `\w` : 単語文字にマッチ
- `\.` : ピリオドにマッチ（\. でエスケープ）
- `\*` : アスタリスクにマッチ
```

---

### プログラミングでの説明

```markdown
## 特殊文字

- 改行: `\n`
- タブ: `\t`
- バックスラッシュ: `\\`
- クォート: `\"` または `\'`
```

---

## コードブロック内ではエスケープ不要

コードブロック（` ``` `）内では、記号はそのまま表示されます。

```markdown
# これは見出しにならない
*これは斜体にならない*
- これはリストにならない
```

インラインコード（`` ` ``）内も同様：

`*テキスト*` ← 斜体にならない

---

## 特殊なケース

### バッククォートをインラインコードで表示

バッククォート1つを表示するには、2つのバッククォートで囲みます。

```markdown
`` ` `` ← バッククォート1つ
```

表示：`` ` ``

### 複数のバッククォート

```markdown
``` `` ``` ← バッククォート2つ
```

---

## エスケープが不要な場合

以下の場合は自動的にそのまま表示されます。

1. **コードブロック内**: ` ``` ` で囲まれた中
2. **インラインコード内**: `` ` `` で囲まれた中
3. **文脈上明らか**: 単独の `*` など、ペアがない場合

```markdown
5 * 3 = 15  ← 単独の * はそのまま表示
a_b_c       ← 文中の _ は環境による
```

---

## 注意点

1. **環境差異**: 一部の記号はエスケープが必要かどうか環境による
2. **過剰エスケープ**: 不要なエスケープは可読性を下げる
3. **コードブロック推奨**: 記号が多い場合はコードブロックを使う方が簡単
