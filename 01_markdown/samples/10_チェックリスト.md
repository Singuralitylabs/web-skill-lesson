# チェックリスト（Task Lists）サンプル

## 解説

タスクの完了状態を表現できるリストです。
GitHubのIssueやPull Requestでよく使われます。

| 記法 | 状態 |
|------|------|
| `- [ ]` | 未完了（チェックなし） |
| `- [x]` | 完了（チェックあり） |

---

## 基本的なチェックリスト

### 記法

```markdown
- [x] 企画書の作成
- [x] レビュー依頼
- [ ] 修正対応
- [ ] 最終提出
```

### 表示結果

- [x] 企画書の作成
- [x] レビュー依頼
- [ ] 修正対応
- [ ] 最終提出

## 入れ子のチェックリスト

### 記法

```markdown
- [ ] プロジェクトA
  - [x] 要件定義
  - [x] 設計
  - [ ] 実装
  - [ ] テスト
- [ ] プロジェクトB
  - [x] 要件定義
  - [ ] 設計
  - [ ] 実装
```

### 表示結果

- [ ] プロジェクトA
  - [x] 要件定義
  - [x] 設計
  - [ ] 実装
  - [ ] テスト
- [ ] プロジェクトB
  - [x] 要件定義
  - [ ] 設計
  - [ ] 実装

## 他の記法との組み合わせ

### 記法

```markdown
- [x] **重要**：セキュリティレビュー完了
- [ ] *任意*：パフォーマンス最適化
- [x] `config.json` の更新
- [ ] [ドキュメント](https://example.com)の確認
```

### 表示結果

- [x] **重要**：セキュリティレビュー完了
- [ ] *任意*：パフォーマンス最適化
- [x] `config.json` の更新
- [ ] [ドキュメント](https://example.com)の確認

---

## ポイント

- `[ ]` の中のスペースは必須です（`[]` ではNG）
- 完了は小文字の `x`（大文字 `X` でも動作する場合あり）
- GitHubでは、Issue/PR内でクリックしてチェックを切り替え可能
- 一部のエディタ（Obsidian等）でもクリックで切り替え可能
