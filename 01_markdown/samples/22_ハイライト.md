# ハイライト（Highlight）サンプル

## 解説

テキストを蛍光ペンでマークしたように強調表示します。重要な箇所を視覚的に目立たせたいときに使います。

| 記法 | 対応環境 | 代替方法 |
|------|----------|----------|
| `==text==` | Obsidian、一部エディタ | `<mark>` タグ |

**注意**：GitHubやVSCodeでは `==` 記法は対応していません。

---

## 基本的なハイライト

### 記法

```markdown
これは==重要なテキスト==です。

==複数の単語をハイライト==することもできます。
```

### 表示結果（対応環境の場合）

これは==重要なテキスト==です。

==複数の単語をハイライト==することもできます。

---

## HTMLタグを使った代替方法

GitHubなど `==` 非対応の環境では `<mark>` タグを使用します。

### 記法

```html
これは<mark>重要なテキスト</mark>です。

<mark>複数の単語をハイライト</mark>することもできます。
```

### 表示結果

これは<mark>重要なテキスト</mark>です。

<mark>複数の単語をハイライト</mark>することもできます。

---

## 他の強調との組み合わせ

### 記法

```markdown
==**太字とハイライト**==

==*斜体とハイライト*==

<mark>**太字**と<mark>ハイライト</mark>の組み合わせ</mark>
```

---

## 実用例

### ドキュメントでの注意喚起

```markdown
## セットアップ手順

1. リポジトリをクローン
2. <mark>必ず `.env` ファイルを作成してください</mark>
3. `npm install` を実行
```

### 変更点の強調

```markdown
## 更新履歴

- v2.0.0: <mark>APIのエンドポイントが変更されました</mark>
- v1.5.0: パフォーマンス改善
```

### 用語の強調

```markdown
Gitでは<mark>ブランチ</mark>を使って並行開発を行います。
```

## 対応環境

| 環境 | `==text==` | `<mark>` タグ |
|------|:----------:|:-------------:|
| Obsidian | ✅ | ✅ |
| GitHub | ❌ | ✅ |
| VSCode プレビュー | ❌ | ✅ |
| Notion | ❌ | ❌ |
| Marked 2 | ✅ | ✅ |

## 実習

以下のテンプレートを使って、重要な部分をハイライトしてみましょう。

```markdown
## 今日の学習メモ

今日は<mark>マークダウンのハイライト記法</mark>を学びました。

重要なポイント：
- <mark>GitHubでは `==` が使えない</mark>
- 代わりに `<mark>` タグを使う
- Obsidianなら `==` が使える
```
