# 脚注（Footnotes）サンプル

## 解説

本文中に注釈や参照を付けられる記法です。
脚注の内容はページ末尾に自動的にまとめて表示されます。

**対応環境**: GitHub、Obsidian、多くのMarkdownエディタ

---

## 基本的な脚注

### 記法

```markdown
マークダウンは2004年に開発されました[^1]。

[^1]: ジョン・グルーバーとアーロン・スワーツによって開発されました。
```

### 表示結果

マークダウンは2004年に開発されました[^1]。

[^1]: ジョン・グルーバーとアーロン・スワーツによって開発されました。

---

## 名前付き脚注

数字の代わりに名前を使うこともできます。

### 記法

```markdown
生成AIの出力はMarkdown形式です[^ai]。
GitHubでもMarkdownが標準です[^github]。

[^ai]: ChatGPT、Claude、Geminiなどが該当します。
[^github]: README.mdやIssue、Pull Requestなどで使用されます。
```

### 表示結果

生成AIの出力はMarkdown形式です[^ai]。
GitHubでもMarkdownが標準です[^github]。

[^ai]: ChatGPT、Claude、Geminiなどが該当します。
[^github]: README.mdやIssue、Pull Requestなどで使用されます。

---

## 複数行の脚注

インデントを揃えることで複数行の脚注が書けます。

### 記法

```markdown
この機能は複雑な処理を行います[^complex]。

[^complex]: この処理は以下のステップで構成されています：
    1. データの検証
    2. 変換処理
    3. 結果の出力

    詳細はドキュメントを参照してください。
```

### 表示結果

この機能は複雑な処理を行います[^complex]。

[^complex]: この処理は以下のステップで構成されています：
    1. データの検証
    2. 変換処理
    3. 結果の出力

    詳細はドキュメントを参照してください。

---

## インライン脚注

一部の環境では、より短い書き方もできます。

### 記法

```markdown
これはインライン脚注です^[脚注の内容をここに直接書けます]。
```

**注意**: この記法はGitHubでは対応していません。

---

## 実用例

### 学術的な文章

```markdown
# 研究レポート

気候変動の影響は年々深刻化しています[^ipcc]。
特に海面上昇は沿岸部に大きな影響を与えます[^sea-level]。

最新の研究では、2050年までに1.5度の上昇が予測されています[^forecast]。

---

## 参考文献

[^ipcc]: IPCC第6次評価報告書（2021年）
[^sea-level]: 国土交通省「海面上昇の影響評価」（2023年）
[^forecast]: Nature Climate Change, Vol.15, 2024
```

---

### 技術ドキュメント

```markdown
## API仕様

このAPIはREST形式です[^rest]。
認証にはOAuth 2.0を使用します[^oauth]。

レート制限は1分間に60リクエストです[^rate]。

[^rest]: REpresentational State Transferの略。
[^oauth]: Open Authorizationの略。業界標準の認可プロトコル。
[^rate]: プランによって上限が異なります。詳細は料金ページを参照。
```

---

### 用語の補足説明

```markdown
このプロジェクトではCI/CD[^cicd]を導入しています。
デプロイにはDocker[^docker]を使用します。

[^cicd]: Continuous Integration / Continuous Delivery（継続的インテグレーション/継続的デリバリー）の略。
[^docker]: コンテナ型の仮想化プラットフォーム。
```

## 脚注の配置について

- 脚注の定義（`[^name]: 内容`）は文書内のどこに書いてもOK
- レンダリング時に自動的にページ末尾にまとめて表示される
- 番号は出現順に自動で振り直される

### 推奨パターン

```markdown
# ドキュメント

本文で脚注を使います[^1]。
別の脚注もあります[^2]。

（本文が続く...）

---

## 脚注

[^1]: 1つ目の脚注
[^2]: 2つ目の脚注
```

---

## 注意点

1. **脚注IDは一意**: 同じIDを複数回定義しない
2. **参照と定義はセット**: 定義だけ、参照だけでは表示されない
3. **環境依存**: Notionなど一部の環境では非対応

---

## 実習

以下のテンプレートを使って、脚注付きの文章を書いてみましょう。

```markdown
# 私の学習メモ

今日は〇〇について学びました[^1]。
特に△△が印象的でした[^2]。

[^1]: 参考にしたサイトや本の情報
[^2]: 補足説明
```
