# マークダウン基礎講座

## はじめに

### マークダウンとは？

マークダウン（Markdown）は、テキストに簡単な記号を加えるだけで、見出しやリストなどの書式を表現できる軽量マークアップ言語です。

2004年にジョン・グルーバーによって開発され、現在では世界中で広く使われています。

### なぜマークダウンを学ぶのか？

- **生成AIの標準出力形式**：ChatGPTやClaudeなどの生成AIは、マークダウン形式で回答を出力します
- **プログラミングとの親和性**：GitHubのREADME、技術ドキュメントの標準形式です
- **シンプルで覚えやすい**：数個の記号を覚えるだけで、構造化された文書が書けます
- **どこでも使える**：メモアプリ、ドキュメントツール、ブログなど多くの場所で対応しています

## 基本文法

### 1. 見出し（Headings）

`#` の数で見出しレベルを表します。`#` の後には半角スペースを入れます。

```markdown
# 見出し1（最も大きい）
## 見出し2
### 見出し3
#### 見出し4
##### 見出し5
###### 見出し6（最も小さい）
```

**表示結果：**

# 見出し1（最も大きい）
## 見出し2
### 見出し3

**ポイント**
- `#` と見出しテキストの間には半角スペースが必要です
- 通常は見出し1〜3を使うことが多いです
- 見出し1は文書のタイトルに使用します

**✏️ 実習**：`practice.md` に見出し1〜3を書いてみましょう。

---

### 2. 段落と改行（Paragraphs & Line Breaks）

**段落**：空行を入れると新しい段落になります。

```markdown
これは最初の段落です。

これは2番目の段落です。
```

**改行**：行末に半角スペース2つ、または `<br>` を入れます。

```markdown
1行目  
2行目（行末にスペース2つ）

1行目<br>
2行目（brタグを使用）
```

**ポイント**
- 単に改行キーを押しただけでは改行されません
- 空行を入れると段落が分かれます

**✏️ 実習**：2つの段落と、段落内での改行を試してみましょう。

---

### 3. リスト（Lists）

**箇条書きリスト**：`-`、`*`、`+` のいずれかを使います。

```markdown
- りんご
- みかん
- ぶどう
```

**表示結果：**
- りんご
- みかん
- ぶどう

**番号付きリスト**：`1.` のように数字とピリオドを使います。

```markdown
1. 最初の手順
2. 次の手順
3. 最後の手順
```

**表示結果：**
1. 最初の手順
2. 次の手順
3. 最後の手順

**入れ子のリスト**：インデント（半角スペース2〜4つ）で階層を作れます。

```markdown
- フルーツ
  - りんご
  - みかん
- 野菜
  - にんじん
  - キャベツ
```

**表示結果：**
- フルーツ
  - りんご
  - みかん
- 野菜
  - にんじん
  - キャベツ

**✏️ 実習**：好きな食べ物を箇条書きで3つ、作り方の手順を番号付きで3つ書いてみましょう。

---

### 4. 強調（Emphasis）

**太字**：`**テキスト**` または `__テキスト__`

```markdown
これは**太字**です。
```

**表示結果：** これは**太字**です。

**斜体**：`*テキスト*` または `_テキスト_`

```markdown
これは*斜体*です。
```

**表示結果：** これは*斜体*です。

**太字かつ斜体**：`***テキスト***`

```markdown
これは***太字かつ斜体***です。
```

**表示結果：** これは***太字かつ斜体***です。

**打ち消し線**：`~~テキスト~~`

```markdown
これは~~打ち消し~~です。
```

**表示結果：** これは~~打ち消し~~です。

**ポイント**
- 日本語文書では斜体が読みにくいため、太字を使うことが多いです
- 生成AIの出力では太字（`**`）がよく使われます

**✏️ 実習**：太字と打ち消し線を使った文を書いてみましょう。

---

### 5. コードブロック（Code Blocks）

**インラインコード**：バッククォート（`` ` ``）で囲みます。

```markdown
`console.log()` は出力に使います。
```

**表示結果：** `console.log()` は出力に使います。

**コードブロック**：バッククォート3つで囲みます。言語名を指定すると構文ハイライトされます。

````markdown
```javascript
function hello() {
  console.log("Hello, World!");
}
```
````

**表示結果：**
```javascript
function hello() {
  console.log("Hello, World!");
}
```

**よく使う言語指定：**
- `javascript` / `js`
- `html`
- `css`
- `python`
- `bash` / `shell`
- `json`

**ポイント**
- 生成AIにコードを出力してもらうと、コードブロック形式で返ってきます
- 言語を指定すると、色分け（シンタックスハイライト）されて読みやすくなります

**✏️ 実習**：インラインコードとコードブロックをそれぞれ書いてみましょう。

---

### 6. リンク（Links）

**基本形式**：`[表示テキスト](URL)`

```markdown
[Google](https://www.google.com)
```

**表示結果：** [Google](https://www.google.com)

**タイトル付きリンク**：マウスオーバーで表示されるタイトルを追加できます。

```markdown
[Google](https://www.google.com "検索エンジン")
```

**URLをそのまま表示**：`<URL>` で囲みます。

```markdown
<https://www.google.com>
```

**表示結果：** <https://www.google.com>

**✏️ 実習**：好きなWebサイトへのリンクを作成してみましょう。

---

### 7. 引用（Blockquotes）

`>` を行頭に付けます。

```markdown
> これは引用文です。
> 複数行にわたる引用も可能です。
```

**表示結果：**
> これは引用文です。
> 複数行にわたる引用も可能です。

**入れ子の引用**：`>` を重ねます。

```markdown
> 外側の引用
>> 内側の引用
```

**表示結果：**
> 外側の引用
>> 内側の引用

**ポイント**
- 他の人の発言や、参照元の文章を示すときに使います
- 生成AIの出力で、ユーザーの質問を引用するときにも使われます

**✏️ 実習**：好きな名言を引用形式で書いてみましょう。

---

### 8. 表（Tables）

パイプ `|` とハイフン `-` を使って表を作ります。

```markdown
| 名前 | 年齢 | 職業 |
|------|------|------|
| 田中 | 25 | エンジニア |
| 鈴木 | 30 | デザイナー |
| 佐藤 | 28 | マーケター |
```

**表示結果：**

| 名前 | 年齢 | 職業 |
|------|------|------|
| 田中 | 25 | エンジニア |
| 鈴木 | 30 | デザイナー |
| 佐藤 | 28 | マーケター |

**配置の指定**：

```markdown
| 左寄せ | 中央寄せ | 右寄せ |
|:-------|:--------:|-------:|
| Left   | Center   | Right  |
```

**表示結果：**

| 左寄せ | 中央寄せ | 右寄せ |
|:-------|:--------:|-------:|
| Left   | Center   | Right  |

**ポイント**
- 2行目のハイフンの数は3つ以上あればOKです
- `:` の位置で配置を指定します（`:---` 左、`:---:` 中央、`---:` 右）
- 生成AIに比較表を出力してもらうとき、この形式になります

**✏️ 実習**：3列×3行の表を作成してみましょう。

---

### 9. 水平線（Horizontal Rules）

`---`、`***`、`___` のいずれかを使います（3つ以上）。

```markdown
上のセクション

---

下のセクション
```

**表示結果：**

上のセクション

---

下のセクション

**✏️ 実習**：水平線でセクションを区切ってみましょう。

---

### 10. チェックリスト（Task Lists）

`- [ ]` で未完了、`- [x]` で完了を表します。

```markdown
- [x] 企画書の作成
- [x] レビュー依頼
- [ ] 修正対応
- [ ] 最終提出
```

**表示結果：**
- [x] 企画書の作成
- [x] レビュー依頼
- [ ] 修正対応
- [ ] 最終提出

**ポイント**
- GitHubのIssueやPull Requestでよく使われます
- 一部のマークダウンエディタではクリックでチェックを切り替えられます

**✏️ 実習**：今日やることリストをチェックリスト形式で書いてみましょう。

---

### 11. 画像（Images）

リンクの先頭に `!` を付けます。

```markdown
![代替テキスト](画像のURL)
```

**例：**
```markdown
![サンプル画像](https://via.placeholder.com/150)
```

**ポイント**
- 代替テキスト（alt属性）は画像が表示できないときに表示されます
- アクセシビリティのためにも代替テキストは必ず入れましょう

## 実践演習：自己紹介ドキュメント

学んだ文法を使って、自己紹介ドキュメントを作成しましょう。

### 要件

以下の要素を含めてください。

1. **見出し**：タイトルと2〜3個のセクション
2. **リスト**：趣味や好きなものを箇条書きで
3. **強調**：重要な部分を太字に
4. **リンク**：SNSや好きなWebサイトへのリンク
5. **コードブロック**（任意）：使える言語があれば

### テンプレート

```markdown
# 自己紹介

## 基本情報

名前：**〇〇**

## 趣味・好きなこと

- 趣味1
- 趣味2
- 趣味3

## スキル

| スキル | レベル |
|--------|--------|
| HTML/CSS | 初級 |
| JavaScript | 勉強中 |

## リンク

- [GitHub](https://github.com/yourname)
- [Twitter](https://twitter.com/yourname)

---

最終更新：2025年1月
```

**✏️ 実習**：10分間で自己紹介ドキュメントを完成させましょう！

## まとめ

本記事では、Markdownの基本文法11種類を解説しました。

Markdownは記法の数が少なく、覚えることもシンプルです。この記事で紹介した内容をマスターすれば、**READMEファイル、技術ドキュメント、ブログ記事、議事録など、実務で必要なほとんどのMarkdown文書を書くことができます**。

生成AI（ChatGPT、Claudeなど）の出力もMarkdown形式です。Markdownを理解していれば、AIの回答をより正確に読み取り、自分でも同じ形式で整理された文書を作成できるようになります。

まずは見出し・リスト・強調の3つから使い始めて、必要に応じて他の記法を取り入れていきましょう。
